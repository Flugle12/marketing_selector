1.1 Описание
Python-приложение для поддержки принятия решений в маркетинге. Система оценивает эффективность рекламных каналов (SEO, SMM, Ads, Email) с помощью Метода анализа иерархий (АИИ / AHP) и определяет оптимальное распределение бюджета в условиях конкуренции, используя аппарат матричных игр (Теория игр). 
Цель: автоматизация выбора стратегии продвижения с математической проверкой согласованности экспертных оценок и поиском Равновесия Нэша.

1.2 Математическая модель
Компонент
	
Описание
	
Реализация
Метод Анализа Иерархий (АИИ)
	
Попарные сравнения каналов, расчет приоритетов (собственный вектор матрицы), вычисление индекса согласованности (CR) для валидации эксперта.
	
core/ahp_solver.py
Матричная игра
	
Антагонистическая игра с конкурентом. Поиск седловой точки (чистые стратегии) и расчет смешанных стратегий (Равновесие Нэша) через решение задачи ЛП.
	
core/game_solver.py
Сценарное моделирование
	
Загрузка YAML-конфигов с тремя сценариями: штатный (CR < 0.1), пограничный (CR ≈ 0.1), стрессовый (CR >> 0.1). Валидация входных данных.
	
config/*.yaml, ui/app.py

1.3 Технологический стек

    Ядро: NumPy (матричные вычисления), SciPy (решение задач линейного программирования для поиска смешанных стратегий).
    Интерфейс: Streamlit (интерактивный веб-дашборд с состоянием session_state).
    Инфраструктура: PyYAML (загрузка сценариев), Pandas (табличное представление результатов).
    SE-практики: pytest, pytest-cov (покрытие ≥ 60%), ruff, black.

1.4 Быстрый старт (Инструкция для проверяющего)
1. Клонирование и настройка окружения:

git clone https://github.com/[user]/marketing_selector.git
cd marketing_selector
python -m venv .venv
source .venv/bin/activate  # Для Windows: .venv\Scripts\activate
pip install -r requirements.txt

2. Запуск веб-интерфейса:

streamlit run ui/app.py

После выполнения команды приложение автоматически откроется в браузере по адресу http://localhost:8501.
3. Демонстрация сценариев (как проверять):

    В боковой панели (Sidebar) выберите сценарий из выпадающего списка.
    Нажмите кнопку «Загрузить сценарий». Матрицы автоматически заполнятся в интерфейсе.
    Нажмите «Рассчитать АИИ» и «Найти Равновесие Нэша».
        Сценарий А (Happy Path): CR ≈ 0.036 находится смешанное равновесие.
        Сценарий Б (Borderline): CR ≈ 0.08 игра имеет седловую точку (чистое равновесие).
        Сценарий В (Stress): CR > 0.1 сложная игра 4×4.
